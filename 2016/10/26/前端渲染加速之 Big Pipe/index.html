前端渲染加速之 - Big Pipe

by 黄乔森 on 2016-10-26

分类 javascript

#前言

首屏渲染速度一直是前端的一个痛点

从最一开的是直接静态资源服务器直接返回资源文件, 到使用CDN分发文件, 再到服务器渲染技术. 无一不是为了让用户得到最佳体验.

Big Pipe 是Facebook所采用的一项加速首屏加载的技术, 在Facebook的首页上可以明显感觉到.

#简介

看起来似乎和Ajax一样呢

首先我们需要知道Ajax其实只是另一个普通的HTTP请求.
一个完整HTTP请求的过程为

DNS Resolving -> TCP handshake -> HTTP Request -> Server -> HTTP Response -> TCP shutdown

整个网络链路上花掉的时间都已经不少.

而 Big Pipe 只需要使用一个连接, 不需要额外的请求

Big Pipe 背后所用的到的技术其实并不复杂, 服务器传给浏览器一个没有闭合的 <body>, 此时浏览器会把已经接收到的dom渲染出来(如果还有css, 也渲染上). 但是, 此时由于TCP连接还没有断开, <body> 还没有闭合, 服务器可以继续推送更多的dom到浏览器, 甚至 <script>.

这样浏览器可以将一个没有数据的页面(相对应的数据展示模块显示加载中占位), 与此同时去从数据库拿数据, 拿到后通过推送<script>标签, 并将数据放入其中. 这样浏览器接收到后, 就可以替换相对应位置的数据了.

##与服务器渲染的区别

服务器渲染与 Big Pipe 有不少相似的地方, 同样是由服务器获取数据, 将其填充到网页DOM中, 返回给客户. 但最大的区别是, Big Pipe 可以在拿到数据之前, 返回一个页面给用户, 减少等待时间, 以防拿数据的操作阻塞太久, 而一直留着一个空白的页面给用户.
